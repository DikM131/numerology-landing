# Глава 1. История проекта

> *«Любой хороший проект начинается не с кода, а с идеи.»*

------------------------------------------------------------------------

# Введение

Проект **Numerolog** родился не как учебный эксперимент и не как попытка
изучить React. Его цель с самого начала была практической --- создать
современный коммерческий лендинг для специалиста по нумерологии, который
выглядел бы дорого, вызывал доверие и мотивировал посетителя записаться
на консультацию.

Изначально была поставлена задача отказаться от типичных клише
эзотерических сайтов. Вместо фиолетовых градиентов, блестящих звёзд,
магических символов и перегруженных декоративных элементов было решено
построить совершенно другую визуальную концепцию.

Основная идея проекта звучала просто:

> **Минимализм. Премиальность. Доверие. Персонализация.**

------------------------------------------------------------------------

# От идеи к проекту

Проект начинался с простого вопроса:

> «Можно ли создать лендинг, который будет выглядеть как сайт дорогого
> бренда, но при этом сохранит атмосферу эзотерики?»

Ответом стала концепция **«Премиальная эзотерика»**.

Она строится на нескольких принципах:

-   тёмный фон вместо ярких космических изображений;
-   благородные золотые акценты вместо кислотных цветов;
-   большое количество свободного пространства;
-   строгая типографика;
-   современные анимации без избыточных эффектов;
-   ощущение спокойствия и уверенности.

------------------------------------------------------------------------

# Почему выбран формат лендинга

На старте проекта основной задачей была не разработка большого сайта, а
создание страницы, которая проводит пользователя по одному сценарию:

1.  заинтересовать;
2.  вызвать доверие;
3.  дать небольшую персонализацию;
4.  показать ценность консультации;
5.  получить заявку.

Для такой задачи одностраничный лендинг оказался оптимальным решением.

------------------------------------------------------------------------

# Первые технологические решения

После обсуждения различных вариантов был выбран следующий стек:

-   React;
-   Vite;
-   Tailwind CSS;
-   Framer Motion;
-   Lucide React;
-   Git + GitHub.

Главная причина такого выбора --- возможность развивать проект без
полной переработки архитектуры.

------------------------------------------------------------------------

# Работа с искусственным интеллектом

Особенностью проекта стало использование двух инструментов ИИ с разными
ролями.

## OpenCode

Использовался как инструмент генерации и изменения кода. Он быстро
реализовывал компоненты, стили и интерфейсные решения.

## ChatGPT

Использовался как архитектор проекта. Его задача заключалась не в
написании максимального количества кода, а в анализе решений,
проектировании архитектуры, разработке дизайн-системы, подготовке
технической документации и формировании качественных промптов для
OpenCode.

Именно разделение ролей позволило значительно повысить качество проекта.

------------------------------------------------------------------------

# Эволюция дизайна

За время разработки проект прошёл несколько этапов.

## Версия 1

Рабочий прототип с базовой структурой страницы.

## Версия 2

Формирование премиальной дизайн-концепции, улучшение Hero-блока и
цветовой палитры.

## Версия 3

Появление стеклянных карточек, фоновой сферы, декоративных колец, частиц
и единой анимационной системы.

## Версия 4

Полировка композиции, вертикального ритма, типографики и Footer. После
этого проект приблизился к состоянию, в котором дальнейшие изменения
стали давать минимальный визуальный эффект.

------------------------------------------------------------------------

# Главный вывод первой главы

Этот проект стал не просто лендингом, а полноценной площадкой для
изучения современных подходов к разработке интерфейсов.

Во время работы были освоены:

-   Git и GitHub;
-   современный стек React;
-   организация компонентов;
-   работа с Tailwind CSS;
-   создание премиальной дизайн-системы;
-   взаимодействие с OpenCode через тщательно подготовленные промпты;
-   совместная работа человека и искусственного интеллекта.

Самым важным результатом стала не отдельная технология, а понимание
процесса создания качественного продукта: от идеи и архитектуры до
финальной полировки интерфейса.

------------------------------------------------------------------------

# Практический вывод

Если бы проект начинался заново сегодня, архитектура и стек технологий
остались бы практически неизменными. Основные усилия по-прежнему были бы
направлены не на поиск новых эффектов, а на создание цельной
дизайн-системы, качественного пользовательского опыта и удобной базы для
дальнейшего развития продукта.

Это и станет основной идеей всей книги: хороший проект рождается не из
случайных решений, а из последовательных и осознанных шагов.
